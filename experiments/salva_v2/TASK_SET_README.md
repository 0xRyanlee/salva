# Task Set v1 — Design Notes & Ground Truth Verification Method

`task_set_v1.json` — 18 fixed, reusable business discovery tasks, replacing
the N=1/N=2 methodology weakness of E10/E21/E21c (single-task live
benchmarks whose results could be dominated by provider-of-the-moment state
or query phrasing rather than telling us anything general).

Validated against the real `DiscoveryRequest`/`DiscoveryIntent` Pydantic
schema (`salva_core/schemas.py`) — every task's `objective`/`intent` block
actually constructs a valid request, not just JSON that looks plausible.

## Difficulty tiers (6 tasks each)

1. **single_entity** — "find X's official website and contact info." The
   simplest tier: one real, unambiguous entity, verify Salva can find it at
   all.
2. **cross_language** — a Chinese company name only in the query
   (`extra_keywords`), ground truth is the correct English legal/investor-
   relations name. Directly targets the known VP5 weakness (Jina embedding
   cosine("台積電", "TSMC") ≈ 0.04 — see `DEVELOPMENT_PROGRESS.md`). Two
   pairs (Chunghwa Telecom, Delta Electronics) include an explicit "trap"
   note about a similarly-named but distinct entity, to test false-positive
   resistance, not just recall.
3. **multi_hop** — "which companies are members/customers/partners of X."
   Tests relationship discovery, not just single-entity lookup. One task
   (`multihop-03-naturehike-dach`) is **deliberately low-confidence/thin-
   evidence** — see below.

## Ground truth verification method

Every `ground_truth_entities` entry was verified via live WebSearch (three
parallel research passes, one per tier, 2026-07-03) against a real, currently
-accessible source URL — not generated by an LLM and accepted without
verification, per this card's explicit guardrail. Each research pass was
instructed to report honestly when evidence was thin rather than padding out
a target count, and to flag anything it could only confirm at medium
confidence.

Two things fell out of that discipline that are worth knowing before using
this task set:

1. **`multihop-03-naturehike-dach` has low confidence by design, not by
   oversight.** No official Naturehike press release or "find a retailer"
   page names authorized DACH distributors — what surfaced were retailer/
   marketplace listings (Amazon.de storefront, a couple of German online
   shops), and no distinct Austrian retailer was found at all. This is kept
   in the task set *on purpose*: it directly mirrors E10/E21's real
   Naturehike DACH dogfood theme, and it's a useful test of whether an agent
   under evaluation also reports "I found thin evidence" honestly instead of
   hallucinating a confident-looking distributor list. When scoring results
   against this task, a good answer that says "coverage is limited, here's
   what I found" should not be penalized the same way a fabricated confident
   list should be — this needs human judgment when Phase 3 scores it, not a
   naive P/R/F1 against the (already low-confidence) ground truth.
2. **A few facts are marked `confidence: medium` inline** (e.g. CNCF's exact
   founding date, GLEIF's exact establishment date) where the official
   source didn't state something directly and a secondary source (Wikipedia,
   an industry knowledge base) was needed. These are still included because
   the *entity identity* itself is high-confidence; only the specific
   secondary fact is softer.

## Why these specific entities/topics

Reused topics from prior experiments deliberately, for before/after
comparability rather than starting from a blank slate each time:

- `multihop-01-cncf-founders` — same topic E21c tested (P=R=F1=0.000 at the
  time; `ddgs` wasn't installed then, see `salva-p1-ddgs-install-verify` and
  `experiments/salva_v2/PROVIDER_ISOLATION_FINDINGS.md`).
- `multihop-03-naturehike-dach` — same real-world theme as E10/E21's
  Naturehike DACH dogfood (`docs/dogfood/naturehike-dach-2026-06-08.md`).
- `multihop-05-gleif-regulators` and the cross-language Taiwan-listed-company
  pairs — same domain as E23 (GLEIF + Wikidata entity resolution).

New entities/topics (MediaTek, Advantech, Delta Electronics, Chunghwa
Telecom, TSMC's customer relationships) were added for breadth beyond what
prior single-task experiments happened to cover, while staying in
recognizable, well-documented B2B/tech-industry territory rather than
inventing obscure or hard-to-verify scenarios.

## How to audit/extend this task set

- To re-verify a ground truth entry: follow its `source_url`(s) directly —
  every entry has at least one.
- To add a new task: follow the same three-part discipline used here —
  (1) design the query/intent, (2) verify ground truth via live search with
  real cited URLs, (3) report honestly if evidence is thin rather than
  padding the answer, and note it in this file the way
  `multihop-03-naturehike-dach` is noted.
- Do not silently edit `ground_truth_entities` without updating the
  `source_urls`/`confidence`/`notes` fields to match — the whole point of
  this file existing is that ground truth changes are auditable, not silent.
- No paid data sources were used or are needed to verify or extend this set
  — free public search and each organization's own official pages are
  sufficient.
